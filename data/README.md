# Data Folder — README

This folder contains all datasets required to replicate the analysis. Files are either generated automatically via the Python data pipeline or compiled manually from public sources, plus a set of additional CSVs that feed directly into the Julia dispatch model.

---

## Folder Structure

```
data/
├── historical_data.csv          # Main hourly dataset (auto-generated)
├── projection_deltas_data.csv   # Monte Carlo delta multipliers by scenario
├── scenario_params.csv          # Scenario-level anomaly parameters
├── technical_params.csv         # Model technical parameters (ramp rates, efficiencies, etc.)
├── technology_data.csv          # Technology-level cost and emissions data
└── other/
    ├── NaturalGas_Daily_Prices2020-2025.xlsx   # Manual — daily gas prices
    ├── Coal_Diesel_ETS_Monthly_Costs.xlsx       # Manual — monthly coal/diesel/ETS prices
    ├── Uranium_Annual_Prices2020-2024.xlsx      # Manual — annual uranium prices
    └── fixed_data_euros.csv                     # Manual — fixed cost estimates
```

## Main Dataset

| File | Description |
|---|---|
| `historical_data.csv` | Final hourly dataset (2020–2024) generated by the pipeline. 43,824 rows × 66 columns. All quantities in GW/GWh; prices in €/GWh. |

This is the file consumed directly by the Julia dispatch model for calibration.

---

## Model Input Files

These four CSVs are read directly by the Julia scripts and do not require any regeneration:

| File | Description |
|---|---|
| `projection_deltas_data.csv` | Delta multipliers for capacity, demand and cost projections to 2030, by variable, year and source. Used to parameterize the Monte Carlo draws. |
| `scenario_params.csv` | Scenario-level anomaly multipliers (elasticity, hydro availability, storage efficiency, etc.) for the four scenarios: Baseline, Nuclear, Optimistic, Climate Change. |
| `technical_params.csv` | Technical parameters for the dispatch model: ramp rates, min/max generation bounds, storage capacities, round-trip efficiencies, grid loss factor. |
| `technology_data.csv` | Technology-level data: heat rates, efficiencies, variable and fixed O&M costs (€/GWh and M€/GW·year), lifecycle and direct CO₂ emission intensities (tCO₂/GWh), fossil fuel flag, and average installed capacity. |

## API-Sourced Data (auto-downloaded)

The following data is fetched automatically when running `Codigo_historical_data.py`. No manual download is needed.

**ESIOS (Red Eléctrica de España)**
- Hourly electricity generation by technology (16 sources)
- Sectoral demand: residential, commercial, industrial
- Hourly spot price (OMIE day-ahead market)
- Net interconnection flows: France, Portugal, Morocco
- Monthly installed capacity by technology (15 sources)
- Self-reported available capacity: hydro, nuclear

**ENTSO-E (European Network of Transmission System Operators)**
- Disaggregated hydroelectric generation: run-of-river, reservoir, and pumped storage (generation and consumption)

To run the pipeline you need valid API credentials for both sources. These are set at the top of `Codigo_historical_data.py`:
```python
TOKEN = "..."           # ESIOS API key — request at api.esios.ree.es
APIKey_entsoe = "..."   # ENTSO-E API key — request at transparency.entsoe.eu
```

## Manually Compiled Files (`other/`)

The following files were compiled manually from public sources and must be present in `data/other/` before running the pipeline. The script will raise a `FileNotFoundError` if any of them is missing.

| File | Frequency | Content | Source |
|---|---|---|---|
| `NaturalGas_Daily_Prices2020-2025.xlsx` | Daily | Natural gas reference price (€/MWh) | MIBGAS |
| `Coal_Diesel_ETS_Monthly_Costs.xlsx` | Monthly | Coal (€/MWh), diesel pre-tax (€/MWh), EU ETS allowance price (€/tCO₂) | World Bank, EEX |
| `Uranium_Annual_Prices2020-2024.xlsx` | Annual | Uranium spot price (€/MWh, converted from USD/lb using a factor of 30.211) | UxC / Cameco |
| `fixed_data_euros.csv` | — | Fixed O&M cost estimates by technology (€/GW·year) | Literature review |

> **Note:** CSV versions of the `.xlsx` files are also accepted by the pipeline. The loader checks for `.xlsx` first and falls back to `.csv` automatically.

## How to Reproduce `historical_data.csv`

1. Make sure the files in `data/other/` described above are present.
2. Set valid API keys in `scripts/Codigo_historical_data.py`.
3. Run the script from the root of the repository:
   ```bash
   python scripts/Codigo_historical_data.py
   ```
4. The script will write `historical_data.csv` to this folder upon completion (~60–90 min depending on API response times).

> The remaining files (`projection_deltas_data.csv`, `scenario_params.csv`, `technical_params.csv`, `technology_data.csv`) are versioned directly in the repository and do not need to be regenerated.

## Missing Data Treatment

Missing values in the raw API downloads are handled inside the pipeline through a three-step imputation strategy applied in strict order:

1. **1-hour gaps** → forward fill from the immediately preceding hour.
2. **Larger blocks** → values copied from exactly 24 hours prior, preserving diurnal seasonality. *Spot prices are excluded from this step and imputed via linear interpolation instead.*
3. **Remaining gaps** → linear interpolation followed by backward fill for edge cases at the start of the series.

After imputation, all physical variables are clipped to ≥ 0.
